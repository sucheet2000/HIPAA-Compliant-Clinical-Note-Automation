About

System Overview
HIPAA-Compliant Clinical Notes Review System

A secure, healthcare-focused application for reviewing and validating clinical note extractions. This system provides a human-in-the-loop interface for clinicians to review AI-generated clinical data with complete audit trails for regulatory compliance.

Key Features
  • De-identification - Automatic masking of Protected Health Information (PHI) before processing
  • Claude AI Integration - Intelligent extraction of clinical entities using Claude API
  • FHIR Standards - Structured data conversion to FHIR R4 format for interoperability
  • Audit Logging - Complete transaction history and compliance trail in PostgreSQL
  • Human Review - Clinician dashboard for validating extractions
  • Confidence Scoring - Field-level confidence metrics for quality assurance

Data Pipeline
  1. Raw clinical conversation input
  2. De-identification module masks PHI
  3. Claude API extracts clinical entities
  4. FHIR transformer converts to standard format
  5. Clinician review and validation
  6. Complete audit trail in PostgreSQL
  7. FHIR bundles stored in MongoDB
Security & Compliance
HIPAA Compliance Features
  • De-identification (Safe Harbor method)
  • Immutable audit logs with PostgreSQL
  • Role-based access control
  • Encrypted data at rest and in transit
  • Transaction correlation for compliance
  • Complete review history tracking
Docker Deployment

The entire system runs in Docker containers with PostgreSQL for audit logs and MongoDB for FHIR data storage. See DOCKER_SETUP.md for deployment instructions.
